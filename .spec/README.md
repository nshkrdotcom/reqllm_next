# `.spec`

This folder is the package-local Spec Led layer for ReqLlmNext.

<!-- spec.workspace.readme_present -->

The repository already has a long-form [`specs/`](../specs) directory for
architecture and refactor notes. `.spec/` is the checked current-truth layer
that drives `mix spec.*` and stays small, operational, and repository-local.

## Canonical Layout

- `README.md` (authored)
- `AGENTS.md` (authored, local operating guidance for agents working in this folder)
- `decisions/README.md` (authored ADR guidance for this workspace)
- `decisions/*.md` (authored durable cross-cutting ADRs)
- `specs/spec_system.spec.md` (authored, meta-spec for this folder)
- `specs/package.spec.md` (authored, package-level intent)
- `specs/*.spec.md` (authored subject specs)
- `state.json` (generated by `mix spec.index` and `mix spec.validate`)

## Core Loop

1. if you are entering the repo or handing work to an agent, run `bw prime`
2. then run `mix spec.prime --base HEAD`
3. make the smallest code, docs, or test change that moves the behavior
4. add or tighten the smallest proof when behavior changed
5. run `mix spec.next`
6. if it says `needs subject updates`, update the right current-truth subject in `specs/`
7. if it says `needs decision update`, add or revise an ADR only when the change is cross-cutting and durable
8. when it says `ready for check`, run `mix spec.check --base HEAD`
9. use `mix spec.status` when you need coverage or frontier summaries
